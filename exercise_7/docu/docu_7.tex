\begin{homeworkProblem}[n-Body - GPU Implementation]
Im Zuge der siebten Übung sollte eine GPU Version des n-Body Problems implementiert werden. 
Zunächst wurde eine naive, unoptimierte Version realisiert. Anschließend ist
diese Version mithilfe von Tiling optimiert worden.

Um eine möglichst hohe Leistung zu erzielen wurde zunächst die optimale Anzahl an Threads
Per Block (tpb) ermittelt. Dafür wurde bei gleichbleibender Problemgröße die tpb variiert.
\begin{center}
\includegraphics[width=0.5\columnwidth]{var_tpb.png}
\end{center}
Es hat sich gezeigt, dass die Performance ab 68 tpb bei 64.000 Bodies relativ konstant
bleiben. Da die optimierte Variante ihren Peak bei 68 tpb hat, wurde dieser Wert für alle
weiteren Messungen verwendet.
\\\\
Für die Ermittlung der Performance wurden die GFLOP/s für eine variierende Bodieanzahl
gemessen.
\begin{center}
\includegraphics[width=0.8\columnwidth]{performance.png}
\end{center}
Der Plot zeigt die verbesserte Performance der optimierten Version. Für eine Bodieanzahl
ab 100.000 können so beinahe 50 GFLOP/s mehr erreicht werden. Für die Schwankungen 
zwischen 16.000 und 70.000 Elementen wurde keine Erklärung gefunden.
\\\\
Abschließend wurde noch der Speed-Up der optimierten Version im Vergleich zur unoptimierten
Version ermittelt.
\begin{center}
\includegraphics[width=0.5\columnwidth]{speed-up.png}
\end{center}
Trotz leichter Schwankungen lässt sich immer eine Verbesserung sehen. Das Ergebniss der
Optimierung ist eine Leistungssteigerung zwischen 5\% - 11\%.
\end{homeworkProblem}
